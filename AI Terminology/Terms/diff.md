## 정의

두 파일이나 텍스트 간의 차이점을 비교하여 어떤 부분이 추가, 삭제, 변경되었는지를 나타내는 결과물이다. 보통 줄(line) 단위로 비교하며, 변경 사항을 구조화된 형식으로 표현한다.

## 왜 중요한가

머신러닝 프로젝트에서 코드 리뷰, 모델 체크포인트 비교, 데이터셋 변경 추적 등을 위해 필수적이다. 특히 버전 관리(version control) 시스템에서 팀 협업의 기반이 되며, 무엇이 언제 왜 변경되었는지 추적할 수 있게 해준다.

## 관련 개념

- 상위 개념: [[Version-Control]](버전 관리)

- 하위 개념: [[Git]](깃), [[Patch]](패치)

- 연관 개념: [[Git-Merge]](깃 병합), [[Code-Review]](코드 리뷰), [[Commit]](커밋)

## 비유로 이해하기

두 버전의 논문을 비교할 때 첫 번째 논문과 두 번째 논문 사이에서 어떤 문단이 추가되었고, 어떤 문단이 삭제되었으며, 어떤 부분이 수정되었는지 표시해주는 것과 같다.

## 실제 사용 예시

`git diff main develop`를 실행하면 main 브랜치와 develop 브랜치 간의 모든 변경사항을 확인할 수 있다. 또는 모델 학습 전후의 설정 파일(config.yaml)을 비교하여 어떤 하이퍼파라미터가 변경되었는지 확인할 수 있다.
