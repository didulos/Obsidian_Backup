## 정의

대규모 데이터 처리 작업에서 전체 데이터를 한 번에 메모리에 로드하지 않고, 작은 단위(블록)로 나누어 순차적으로 처리하는 기법이다. 각 블록은 독립적으로 처리되며, 처리 결과는 누적되거나 저장된다.

## 왜 중요한가

제한된 메모리 환경에서 대규모 데이터셋을 효율적으로 처리할 수 있으며, 시스템 안정성을 보장한다. 머신러닝 모델 학습 시 배치 단위 처리(batch processing)의 기초가 되므로 실무에서 필수적이다.

## 비유로 이해하기

큰 책을 한 번에 읽지 않고 장(chapter) 단위로 나누어 읽는 것처럼, 거대한 데이터셋을 작은 블록 단위로 나누어 처리한다.

## 실제 사용 예시

신경망 학습 시 10GB 데이터셋을 32개 샘플씩 묶은 블록 단위로 나누어 처리하면 메모리 오버플로우를 방지할 수 있다. 데이터베이스에서 수백만 개의 레코드를 읽을 때 1만 개씩 블록으로 나누어 처리한다.
