## 정의

Hermes-Kanban은 머신러닝 워크플로우와 데이터 파이프라인의 진행 상황을 시각적으로 관리하는 칸반(Kanban) 기반의 프로젝트 관리 방식이다. 실험, 데이터 처리, 모델 학습 등 ML 프로젝트의 각 단계를 '할 일(To-Do)', '진행 중(In Progress)', '완료(Done)' 등의 상태로 구분하여 추적한다.

## 왜 중요한가

ML 프로젝트는 반복적인 실험과 데이터 처리로 복잡도가 높기 때문에, 팀 내 진행 상황 공유와 병목 지점 파악이 필수적이다. 칸반 방식을 통해 작업 흐름을 명확히 가시화하면 협업 효율성을 높이고 불필요한 작업 중복을 방지할 수 있다.

## 비유로 이해하기

공장의 조립 라인에서 각 작업 단계(부품 준비, 조립, 품질 검사, 배송)가 칸(Kanban)에 표시되는 것처럼, ML 프로젝트의 각 작업 단계를 카드 형태로 관리하면서 누가 무엇을 하고 있는지 한눈에 파악할 수 있다.

## 실제 사용 예시

데이터 수집 → EDA(탐색적 데이터 분석) → 전처리 → 모델 학습 → 평가의 5단계를 칸반 보드에 배치하고, 각 단계별 진행 상황을 지라(Jira)나 노션(Notion)에서 추적하며 팀원과 공유한다.
