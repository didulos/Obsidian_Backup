## 정의

훈련된 머신러닝 모델을 프로덕션 환경에 배포하여 실제 사용자의 요청을 처리하도록 하는 과정이다. 모델을 API, 웹 서비스, 또는 엣지 디바이스 등 다양한 형태로 운영 환경에 전개하는 것을 의미한다.

## 왜 중요한가

아무리 우수한 모델도 배포되지 않으면 비즈니스 가치를 창출할 수 없다. 배포 과정에서 모델 성능 저하, 버전 관리, 모니터링 등을 적절히 처리하지 않으면 운영 문제가 발생할 수 있다.

## 비유로 이해하기

레스토랑에서 새로운 요리법을 개발해서 테스트하는 것은 모델 훈련이고, 그 요리를 실제 메뉴에 올려 고객에게 서빙하는 것이 배포다.

## 실제 사용 예시

이미지 분류 모델을 Docker 컨테이너로 패키징하여 Kubernetes 클러스터에 배포하고, 로드 밸런서를 통해 REST API 엔드포인트를 제공한다. 추천 모델의 새 버전을 카나리 배포 방식으로 10%의 트래픽부터 점진적으로 늘려가며 검증한다.
