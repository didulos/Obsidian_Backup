## 정의

머신러닝 모델을 구축하고 배포하기 위해 데이터 수집부터 모델 학습, 평가, 배포까지의 일련의 자동화된 단계(stages)를 연결한 워크플로우다. 각 단계의 출력이 다음 단계의 입력으로 자동으로 전달되며, 데이터와 코드의 흐름을 체계적으로 관리한다.

## 왜 중요한가

파이프라인을 구축하면 반복적인 작업을 자동화하여 개발 시간을 단축하고, 일관성 있는 모델 배포를 보장하며, 문제 발생 시 각 단계를 쉽게 추적하고 수정할 수 있다. 특히 프로덕션 환경에서 모델을 지속적으로 관리하고 개선하는 데 필수적이다.

## 비유로 이해하기

자동차 생산 공장의 조립 라인과 같다. 각 스테이션(부품 조립, 용접, 도장, 검사)에서 자동으로 처리되고 다음 스테이션으로 넘어가듯이, ML 파이프라인도 데이터가 각 단계를 거쳐 자동으로 처리되고 최종적으로 완성된 모델이 나온다.

## 실제 사용 예시

scikit-learn의 `Pipeline`을 사용하여 데이터 정규화(normalization) → 특성 선택(feature selection) → 모델 학습을 하나의 객체로 통합하거나, Apache Airflow로 매일 자정에 자동으로 데이터 수집 → 모델 재학습 → 성능 검증 → 배포를 순차적으로 실행한다.
