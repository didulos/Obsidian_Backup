## 정의

버전 관리 시스템(version control system)에서 코드나 데이터의 변경사항을 저장소에 영구적으로 기록하는 행위이다. 각 커밋(commit)은 고유한 식별자(hash)를 가지며, 변경 내용, 작성자, 타임스탐프 등의 메타데이터를 포함한다.

## 왜 중요한가

ML 프로젝트에서 모델 코드, 학습 스크립트, 데이터 처리 로직의 모든 변경사항을 추적할 수 있어 실험 재현성(reproducibility)을 보장한다. 또한 팀 협업 시 누가 언제 무엇을 변경했는지 명확히 할 수 있다.

## 비유로 이해하기

문서 작업 시 "최종_v1.docx", "최종_v2_수정.docx" 같은 파일을 만드는 것처럼, 커밋은 코드의 특정 시점을 스냅샷(snapshot)으로 저장하고 그 변경 내용을 기록하는 것이다.

## 실제 사용 예시

```
git commit -m "모델 하이퍼파라미터 튜닝: learning rate 0.001로 조정"
```

이렇게 커밋하면 해당 변경사항이 저장소 히스토리에 영구 기록되어, 나중에 이전 버전으로 돌아가거나 특정 시점의 코드를 확인할 수 있다.
