## 정의

REST-API는 HTTP 프로토콜을 기반으로 자원(resource)에 접근하기 위한 아키텍처 스타일이다. 클라이언트가 표준 HTTP 메서드(GET, POST, PUT, DELETE)를 사용하여 서버의 데이터를 조회, 생성, 수정, 삭제할 수 있도록 설계되었다.

## 왜 중요한가

REST-API는 웹 서비스 통신의 표준 방식으로, 머신러닝 모델 서빙부터 데이터 파이프라인까지 모든 실무 환경에서 필수적이다. 간단하고 확장 가능한 설계로 다양한 클라이언트와 서버 간 상호작용을 효율적으로 구현할 수 있다.

## 비유로 이해하기

REST-API는 음식점의 주문 시스템과 같다. 손님(클라이언트)이 메뉴판을 보고 "주문하기(POST)", "주문 확인(GET)", "주문 변경(PUT)", "주문 취소(DELETE)"라는 정해진 방식으로 요청하면, 주방(서버)이 그에 맞춰 음식을 준비해서 건네주는 것이다.

## 실제 사용 예시

머신러닝 모델을 서빙할 때 Flask나 FastAPI를 사용해 `GET /predict?input=value` 또는 `POST /predict`로 예측 요청을 받는다. 데이터 파이프라인에서도 `GET /api/v1/data` 방식으로 외부 서비스와 통신한다.
