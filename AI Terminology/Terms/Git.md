## 정의

버전 관리 시스템(version control system)으로, 코드와 파일의 변경 이력을 추적하고 여러 개발자가 협업할 수 있도록 하는 분산형 도구이다. 각 변경사항을 커밋(commit)으로 기록하여 언제든 이전 버전으로 복구할 수 있다.

## 왜 중요한가

ML 프로젝트에서 코드, 모델, 설정 파일의 변경을 체계적으로 관리하고 팀 협업을 가능하게 하며, 문제 발생 시 빠르게 원인을 추적할 수 있다.

## 비유로 이해하기

구글 문서(Google Docs)의 '버전 기록' 기능처럼, Git은 모든 변경사항을 시간 순서대로 기록하여 언제든 과거 상태로 돌아갈 수 있게 해준다.

## 실제 사용 예시

모델 학습 코드를 수정한 후 `git commit -m "improve model accuracy"` 명령으로 변경을 저장하고, 나중에 문제가 발생하면 `git log`로 이전 커밋들을 확인하여 어디서 문제가 생겼는지 추적한다.
