## 정의

에이전트(agent)가 수행해야 할 작업을 더 작은 단위의 부분 작업(subtask)으로 분해하고, 각 부분 작업을 적절한 실행 단위나 워커(worker)에 할당하는 프로세스이다. 이를 통해 복잡한 작업을 병렬로 처리하거나 전문화된 컴포넌트가 각자의 역할을 수행하도록 조율한다.

## 왜 중요한가

자동화 에이전트의 확장성(scalability)과 효율성을 높이는 핵심 메커니즘이다. 작업 분해를 통해 복잡한 문제를 관리 가능한 단위로 나누고, 병렬 처리로 전체 실행 시간을 단축할 수 있다.

## 비유로 이해하기

대형 프로젝트의 프로젝트 매니저가 전체 작업을 여러 팀에 할당하는 것처럼, 에이전트가 큰 목표를 여러 작은 작업으로 나누어 각 팀(또는 서브 에이전트)에 배정하는 것이다.

## 실제 사용 예시

LLM 기반 에이전트가 고객 분석 요청을 받으면, 이를 "데이터 수집", "통계 분석", "리포트 생성" 3개 작업으로 분해하여 각각 다른 함수나 서브 에이전트에 위임한다.
