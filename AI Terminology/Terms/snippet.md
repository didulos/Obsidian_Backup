## 정의

코드나 데이터의 작은 단위 조각 (snippet)을 재사용 가능한 형태로 저장하고 관리하는 방식이다. 머신러닝 실무에서는 전처리 코드, 모델 학습 루틴, 평가 스크립트 등 자주 반복되는 로직을 snippet으로 만들어 프로젝트 간 활용한다.

## 왜 중요한가

반복적인 코드 작성을 줄이고 팀 간 코드 표준화를 유지하며, 버그를 줄인 검증된 로직을 빠르게 재사용할 수 있게 해준다. 특히 데이터 파이프라인 구축 시 개발 속도를 크게 단축할 수 있다.

## 비유로 이해하기

요리할 때 자주 쓰는 레시피 카드를 따로 모아놓고 필요할 때 꺼내 쓰는 것처럼, 자주 사용하는 코드 조각을 저장해두었다가 새 프로젝트에서 필요할 때 바로 적용하는 것이다.

## 실제 사용 예시

데이터 정규화(normalization) 함수를 snippet으로 저장해두면, 여러 프로젝트의 전처리 단계에서 동일한 로직을 반복 구현하지 않고 바로 import 해서 사용할 수 있다. IDE의 snippet 기능(VS Code의 사용자 정의 스니펫)을 활용해 모델 평가 코드 패턴을 자동완성으로 삽입할 수도 있다.
