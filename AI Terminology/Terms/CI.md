## 정의

지속적 통합(Continuous Integration)은 개발자가 작성한 코드 변경사항을 중앙 저장소에 자동으로 빌드, 테스트, 검증하는 개발 관행이다. 코드 변경이 발생할 때마다 자동화된 파이프라인이 실행되어 품질을 즉시 검증한다.

## 왜 중요한가

ML 프로젝트에서 모델 코드, 데이터 처리 로직, 의존성 변경이 자주 발생하는데, CI를 통해 이러한 변경이 즉시 테스트되어 버그를 조기에 발견하고 배포 준비 시간을 단축할 수 있다.

## 비유로 이해하기

CI는 공장의 품질 검사 라인과 같다. 제품(코드)이 생산될 때마다 자동으로 검사되어 불량품(버그)이 다음 단계로 넘어가는 것을 방지한다.

## 실제 사용 예시

GitHub 저장소에 모델 학습 코드를 푸시하면 자동으로 단위 테스트, 코드 품질 검사, 모델 성능 기준(baseline) 검증이 실행되고, 모든 테스트를 통과한 경우에만 병합을 허용한다.
