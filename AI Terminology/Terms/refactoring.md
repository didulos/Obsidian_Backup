## 정의

코드의 기능을 변경하지 않으면서 내부 구조를 개선하는 작업이다. 가독성 향상, 중복 제거, 복잡도 감소 등을 통해 코드 품질을 높인다. 머신러닝 프로젝트에서는 모델 학습 파이프라인, 데이터 처리 로직, 모델 서빙 코드 등을 개선하는 데 적용된다.

## 왜 중요한가

기술부채(technical debt)를 줄이고 코드 유지보수성을 높여 장기적인 개발 속도를 가속화한다. 특히 ML 모델이 프로덕션 환경에서 계속 업데이트되어야 하는 상황에서 리팩토링은 버그 감소와 협업 효율성 개선에 필수적이다.

## 비유로 이해하기

집을 살기 좋게 개조하는 것과 같다. 벽을 허물거나 레이아웃을 바꿔서 생활 공간은 더 넓어지지만, 결국 같은 집이고 주소는 변하지 않는다.

## 실제 사용 예시

데이터 전처리 함수에서 중복된 로직을 별도 함수로 추출하거나, 하드코딩된 하이퍼파라미터를 설정 파일로 분리하는 것이 리팩토링의 예시다.
