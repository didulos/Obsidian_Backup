## 정의

Pull Request(PR)는 버전 관리 시스템에서 한 브랜치(branch)의 코드 변경사항을 다른 브랜치(주로 main 또는 develop)에 병합(merge)하기 전에, 팀원의 검토와 승인을 요청하는 메커니즘이다. 코드 품질 관리와 협업을 위한 필수 프로세스이다.

## 왜 중요한가

PR은 코드 리뷰(code review)를 통해 버그를 조기에 발견하고, 팀의 코딩 표준을 유지하며, 프로덕션 환경으로의 무분별한 배포를 방지한다. 특히 ML 프로젝트에서는 모델 변경사항, 데이터 처리 로직, 학습 파이프라인의 안정성을 검증하는 핵심 게이트(gate)가 된다.

## 비유로 이해하기

PR은 논문 투고 과정과 유사하다. 연구자가 논문을 학회에 제출하면 심사위원들이 검토하고 피드백을 주며, 수정 후 승인받아야 게재되는 것처럼, 개발자가 코드 변경을 PR로 제출하면 팀원들이 검토하고 피드백을 준 후 승인받아야 메인 브랜치에 병합된다.

## 실제 사용 예시

데이터 엔지니어가 특성 추출(feature engineering) 로직을 개선한 코드를 새 브랜치에서 작성 후 PR을 생성하면, 팀원들이 데이터 정합성, 성능 영향도를 검토한 뒤 승인하면 프로덕션 파이프라인에 병합된다.
