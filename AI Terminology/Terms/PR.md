## 정의

PR(Pull Request)은 코드 변경사항을 중앙 저장소(repository)에 병합하기 전에 검토받기 위해 제출하는 요청이다. 팀원들이 코드를 검증하고 피드백을 주는 협업 프로세스의 핵심 메커니즘이다.

## 왜 중요한가

ML/AI 프로젝트에서 모델 코드, 데이터 파이프라인, 실험 결과를 팀이 함께 검토하면서 버그를 사전 차단하고 코드 품질을 유지할 수 있다. 특히 재현성(reproducibility)이 중요한 ML 워크플로우에서 필수적이다.

## 비유로 이해하기

PR은 식당의 신메뉴 개발 과정과 같다. 셰프가 새로운 요리를 만들었을 때, 바로 메뉴에 올리지 않고 먼저 오너나 다른 셰프들에게 맛을 보여주고 의견을 받은 후 개선하여 최종 승인을 받는 것처럼, 코드도 검토 후 병합된다.

## 실제 사용 예시

모델 개선 코드를 feature 브랜치에서 작성한 후 PR을 열면, 팀원들이 모델 성능 지표와 코드 스타일을 검토하고 "이 부분 최적화 가능합니다" 같은 코멘트를 남긴 후 승인(approve)하면 메인 브랜치로 병합된다.
