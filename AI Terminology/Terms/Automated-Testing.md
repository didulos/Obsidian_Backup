## 정의

소프트웨어 테스트 프로세스를 자동화하여 수동 개입 없이 코드 변경사항을 검증하는 기법이다. 단위 테스트(unit test), 통합 테스트(integration test), 엔드-투-엔드 테스트(end-to-end test) 등을 자동으로 실행하고 결과를 평가한다.

## 왜 중요한가

개발 속도를 높이면서도 버그를 조기에 발견하고, 배포 전 코드 품질을 지속적으로 보장한다. 특히 ML 모델 개발에서 데이터와 코드 변경에 따른 예상치 못한 영향을 빠르게 감지하는 데 필수적이다.

## 비유로 이해하기

자동화된 테스트는 공장의 품질 검사 로봇(quality control robot)과 같다. 모든 제품이 나올 때마다 자동으로 규격을 확인하므로 불량품이 시장에 나가기 전에 걸러낸다.

## 실제 사용 예시

pytest로 작성한 테스트 스크립트를 GitHub Actions에서 자동 실행하여 pull request마다 모델 성능 저하 여부를 검증한다. 데이터 파이프라인의 각 단계별 출력 형식과 통계값을 자동으로 검사하여 이상 징후를 조기에 탐지한다.
