## 정의

소프트웨어 개발에서 재사용 가능한 코드, 함수, 모듈을 모아놓은 패키지나 프레임워크(library)를 의미한다. AI/ML 분야에서는 데이터 처리, 모델 학습, 배포를 위한 전문화된 라이브러리들을 활용한다. 개발자가 처음부터 모든 기능을 구현할 필요 없이 검증된 도구를 통해 생산성을 높인다.

## 왜 중요한가

ML 프로젝트에서 핵심 알고리즘과 데이터 처리 파이프라인을 직접 구현하면 시간과 비용이 급증한다. 검증된 라이브러리를 활용하면 버그를 줄이고 성능을 보장받으면서 개발 속도를 크게 향상시킬 수 있다.

## 비유로 이해하기

레고 블록처럼 미리 만들어진 부품들을 조립해서 원하는 구조물을 만드는 것과 같다. 각 부품(라이브러리)은 특정 역할을 하고 있으며, 개발자는 이들을 조합해 전체 시스템을 구축한다.

## 실제 사용 예시

`import numpy as np` 또는 `from sklearn.preprocessing import StandardScaler`를 통해 데이터 변환과 머신러닝 모델링에 필요한 기능들을 즉시 사용할 수 있다. 이러한 라이브러리들이 없다면 행렬 연산이나 정규화 알고리즘을 모두 직접 구현해야 한다.
