## 정의

머신러닝 모델의 학습 및 배포 과정에서 데이터, 코드, 모델 버전 간의 의존성(dependency)을 추적하고 관리하는 엔지니어링 접근법이다. 실험의 재현성(reproducibility)을 보장하고 프로덕션 환경에서 안정적인 모델 운영을 가능하게 한다.

## 왜 중요한가

모델 개발 과정에서 어떤 데이터와 코드로 특정 성능을 낸 모델이 만들어졌는지 명확히 추적할 수 없으면, 문제 발생 시 원인 파악이 어렵고 결과 재현이 불가능해진다. Harness Engineering은 이러한 복잡성을 체계적으로 관리하여 신뢰할 수 있는 ML 시스템 구축을 가능하게 한다.

## 비유로 이해하기

건설 현장에서 건물을 지을 때 어떤 자재를 언제 어디서 사용했는지, 어떤 단계에서 품질 검사를 했는지 모두 기록하는 것처럼, 머신러닝도 모델이 어떤 데이터와 설정으로 만들어졌는지 완벽하게 기록하고 추적하는 것이다.

## 실제 사용 예시

DVC(Data Version Control)를 사용하여 학습 데이터 버전 5.2와 모델 가중치를 함께 저장하면, 3개월 뒤에도 동일한 조건으로 모델을 재현할 수 있다. MLflow에서 하이퍼파라미터, 학습 곡선, 모델 아티팩트를 모두 기록하면 어느 실험이 최고 성능이었는지 추적 가능하다.
