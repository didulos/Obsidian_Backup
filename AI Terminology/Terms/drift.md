## 정의

모델이 학습한 데이터 분포(data distribution)와 실제 운영 환경의 데이터 분포가 달라지면서 모델의 성능이 저하되는 현상이다. 시간이 지나면서 입력 데이터의 특성이 변하거나, 예측 대상의 통계적 성질이 변할 때 발생한다.

## 왜 중요한가

프로덕션 환경에서 모델 성능 저하를 조기에 감지하고 재학습(retraining)을 결정하는 핵심 지표이다. drift를 모니터링하지 않으면 실제로는 부정확한 예측을 하고 있는데도 모르고 운영하게 된다.

## 비유로 이해하기

의사가 특정 환자 그룹으로 질병 진단 모델을 학습했는데, 시간이 지나 환자의 인구통계(나이, 성별 비율)가 크게 바뀌었다면 모델의 진단 정확도가 떨어지는 것과 같다.

## 실제 사용 예시

추천 시스템이 여름에 학습한 상품 선호도 모델이 겨울에 들어서면서 사용자 구매 패턴이 바뀌어 추천 정확도가 하락한다. 이를 감지하기 위해 매주 모니터링 대시보드에서 특성의 통계 분포 변화(statistical test)를 확인한다.
