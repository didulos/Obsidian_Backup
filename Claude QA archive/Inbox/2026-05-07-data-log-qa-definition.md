# 데이터 로그 QA란?

## Ask & Output

**Ask:**
데이터 로그 qa란 무엇인가?

**Output:**
"데이터 로그 QA"는 로그 데이터의 품질 검증(Quality Assurance)을 의미하며, 주로 데이터 엔지니어링/애널리틱스 맥락에서 무결성·스키마·값의 타당성·일관성 4가지 차원에서 점검한다. 새 트래킹 코드 배포, 대시보드 이상치, 신규 파이프라인 검수, A/B 테스트 신뢰성 확인 등의 상황에서 트리거된다.

## Lessons Learned

- 데이터 로그 QA는 단순히 로그를 "들여다보는" 작업이 아니라, **무결성·스키마·값의 타당성·일관성** 4가지 차원으로 나누어 점검하는 체계적 활동이다.
- "QA를 하고 싶다"는 모호한 동기에서 시작하더라도, 트리거 상황(배포 후 검증 / 대시보드 이상치 / 파이프라인 검수 / A/B 테스트)을 먼저 특정하면 구체적인 점검 항목이 자연스럽게 따라온다.

## Better Ask

**Before:**
데이터 로그 qa란 무엇인가?

**After:**
데이터 파이프라인/이벤트 트래킹 맥락에서 "데이터 로그 QA"가 일반적으로 어떤 활동을 의미하는지, 어떤 차원(무결성·스키마·값·일관성 등)으로 분해해서 점검하는지, 그리고 어떤 상황에서 주로 트리거되는지 알려줘. 나는 아직 어떤 종류의 로그를 다룰지 정하지 못한 상태라 일반론 수준의 정의가 필요해.
